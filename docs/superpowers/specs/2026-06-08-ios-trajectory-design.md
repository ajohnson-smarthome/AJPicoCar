# iOS: траектория движения + переработка диаграммы машинки — Design

**Дата:** 2026-06-08
**Статус:** дизайн утверждён (через визуальный компаньон), готов к плану

## Цель

Переработать центральный элемент экрана пульта (диаграмма «вид сверху») в живой
**индикатор движения с предполагаемой траекторией**, как направляющие в парковочных/360-камерах.
Считается из текущей команды `t,y`. Чисто визуально — это **прогноз из команды**, не реальный путь
(одометрии/энкодеров нет).

Заменяет текущий `WheelsView` (стрелка+свечение) на новую диаграмму с анимацией колёс + рельсами.

## Решения (выбраны в визуальном компаньоне)

| Аспект | Решение |
|---|---|
| Машинка | Вид сверху: вытянутый кузов (**~+32%** длины от исходного) + **4 колеса по углам** |
| Колесо (глиф) | **CD-A: бегущие шевроны** — стрелки маршируют вдоль колеса; направление по форме, скорость = темп бега + плотность шевронов |
| Цвет колеса | вперёд — **зелёный** (accent), назад — **янтарный** (`#e0a430`); idle — приглушённый |
| Траектория | **Две направляющие** («рельсы») по ширине машинки, **затухают к концу** (ярко у машинки → прозрачно вдаль), без рисок |
| Старт рельсов | с **зазором** над передними колёсами (вперёд) / под задними (назад) — не заползают на колёса |
| Вперёд (t>0) | рельсы вверх, зелёный; колёса зелёные, шевроны вверх |
| Назад (t<0) | рельсы вниз, янтарный; колёса янтарные, шевроны вниз |
| Изгиб / длина | изгиб по `yaw` (y); длина/яркость рельсов растёт с `|throttle|` |
| Разворот на месте (t≈0, y≠0) | **B+C**: две стрелки `↻` вокруг центра машинки (вращение по знаку y) **+** борта в противоход (один зелёный↑, другой янтарный↓) |

## Геометрия траектории (модель)

Skid-steer: forward-скорость ∝ `t`, угловая ∝ `y`. Путь — дуга окружности кривизны `κ ∝ y/t`.
Рельс рисуем параметрически: идём по дуге длиной `L ∝ |t|`, heading `θ(s)=θ₀+κ·s`, интегрируем
позицию; две направляющие — смещение ±(полуширина машинки) перпендикулярно heading; ~24 точки → `Path`.

Состояния (пороги `EPS_T`, `EPS_Y` ≈ 0.05):
- `|t| ≥ EPS_T` → рельсы (вперёд/назад по знаку t).
- `|t| < EPS_T` и `|y| ≥ EPS_Y` → **разворот** (B+C: дуги-стрелки + борта в противоход).
- обе малы → ни рельсов, ни стрелок (машинка стоит, колёса idle).

## Компонент

`ios/ESP32Car/DriveDiagram.swift` — единый центральный вид, вход `t: Double`, `y: Double`,
`palette: Palette`. Заменяет `WheelsView`. Использует `TimelineView(.animation)` для непрерывной
анимации (бегущие шевроны) + `Canvas` для отрисовки:
- **Машинка:** кузов (вытянутый) + 4 колеса по углам.
- **Колёса (CD-A):** в каждом колесе — бегущие шевроны; направление/скорость борта из
  `ControlModel.sides(t,y)` (левые колёса = left, правые = right); смещение шевронов = `phase` из
  времени, темп ∝ `|скорость|`; цвет зелёный/янтарный по знаку; плотность шевронов растёт с `|скорость|`.
- **Оверлей по состоянию:**
  - движение → две затухающие рельсы (gradient stroke), изгиб по y, длина по |t|, зазор от колёс,
    цвет по знаку t.
  - разворот → две дуги-стрелки `↻` вокруг центра машинки + борта окрашены в противоход.

Чистую геометрию (точки рельса; выбор состояния по порогам) выносим в pure-функции для хост-теста
(`ios/ESP32CarTests`): прямые рельсы при y≈0; изгиб по знаку y; «разворот» при t≈0,y≠0; «пусто».

## Размещение в `DriveView`

Центр: `HStack [L · DriveDiagram(t:curT, y:curY, palette:p) · R]` (вместо `WheelsView`).
L/R-подписи — фикс-ширина 64 (уже сделано). Источник `t,y` — `curT/curY` из `push()`.
`TrajectoryView`/`WheelsView` как отдельные файлы не нужны — всё в `DriveDiagram` (старый
`WheelsView` удаляем).

## Темы

Цвета из `Palette` (тёплая тёмная / тёплая светлая). Янтарный реверса — общий `#e0a430`
(или добавить `palette.warn` для светлой/тёмной при необходимости контраста).

## Тестирование

- **Pure (нативно/симулятор):** выбор состояния по порогам; точки рельса (прямо/изгиб); знак борта→
  цвет/направление шеврона.
- **Симулятор:** двигать стик мышью → рельсы изгибаются/удлиняются, шевроны бегут быстрее;
  реверс → вниз+янтарный; спин (танк-стик врозь) → ↻ + борта в противоход; обе темы.
- **Сборка:** `xcodebuild` под симулятор-SDK.

## Вне объёма

- Реальная траектория/одометрия (нет датчиков) — прогноз из команды.
- Препятствия/коллизии, риски дистанции (отклонены), мини-карта.
- Прошивка не трогается.
