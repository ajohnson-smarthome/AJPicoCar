# Калибровка — редизайн в конвенциях экрана обновления — Design

**Дата:** 2026-06-09
**Статус:** дизайн утверждён (визуальный компаньон), готов к плану

## Цель

Переработать `CalibrationView` под те же конвенции, что у экрана обновления: машинка **1:1 как в прошивке**
(стройный корпус + лобовое + колёса поверх, тот же размер/стиль), но **колёса интерактивные и цветные**;
кольца-радар **пульсируют по процессу**; правый блок — единый шаблон **сегменты · Заголовок · Подзаголовок ·
кнопка(и)** с кастомными пилюлями. Поведение (spin/tap/direction/save) — без изменений.

## Левая картинка — машинка + интерактивные колёса

Силуэт ровно как `FirmwareCarView`: корпус 34×72 (опак, bg-под-panel), лобовое, колёса 11×15 **поверх** корпуса
по углам (offset x ±18.5, y ±20.5), весь блок `scaleEffect(1.9)`, frame ~200×240. **Чипа нет** (фокус — колёса).

**Колёса — `Button` (тап-зона ≥ визуала), цвет по состоянию:**
- **idle** — тёмные (`#3a3024`, как в прошивке), пусто; кликабельны.
- **pending** (нажато, ждём направление) — `warn` (янтарь), пульс свечения.
- **assigned** — `accent` (зелёный) + `✓`.
- **failed-экран** — все 4 = `warn` + `✕`.

**Кольца (как в прошивке, пульс):** пульсируют, когда **ждём/работаем** — на шаге spin (ждём тап) и при
сохранении. Выкл — на направлении / готово / ошибке.

## Правый блок — единый шаблон (сегменты · Заголовок · Подзаголовок · кнопки)

Сегменты-прогресс (4) сверху; ниже `Title` (≈18–22pt semibold) + `Subtitle` (≈12–14pt muted) + пилюли-кнопки
(как `fwButton`: prominent = accent-tint + accent-текст + обводка; amber-вариант для «Назад»).

| Состояние | Кольца | Заголовок | Подзаголовок | Кнопки |
|---|---|---|---|---|
| **spin** (шаг N, ждём тап) | пульс | Шаг N из 4 | Нажми «Крутить», затем колесо, которое поехало. | Крутить (prom) |
| **direction** (нажат `C`) | выкл | Колесо `C` | Куда оно крутилось? | Вперёд (accent) · Назад (amber) |
| **done** (все размечены) | выкл | Готово | Все колёса размечены. | Сохранить (prom) |
| **saving** | пульс | Сохранение… | Записываю калибровку в машинку. | — |
| **failed** | выкл | Не удалось | Проверь связь и повтори. | Повторить (prom) |

Состояние вычисляется из `(saving, errMsg, pending, step)`: `saving → .saving`; `errMsg≠nil → .failed`;
`pending≠nil → .direction`; `step≥4 → .done`; иначе `.spin`.

## Строки (`L` + `ru.lproj`)

Оставить: `calib.title`, `calib.step` («Шаг %d из 4»), `calib.spin` («Крутить»), `calib.forward`, `calib.back`,
`calib.allSet` («Все колёса размечены.»), `calib.save` («Сохранить»). Добавить/заменить:
```
"calib.spinSub"   = "Нажми «Крутить», затем колесо, которое поехало.";
"calib.wheel"     = "Колесо %@";
"calib.whichDir2" = "Куда оно крутилось?";
"calib.doneTitle" = "Готово";
"calib.saving"    = "Сохранение…";
"calib.savingSub" = "Записываю калибровку в машинку.";
"calib.failTitle" = "Не удалось";
"calib.failSub"   = "Проверь связь и повтори.";
"calib.retry"     = "Повторить";
```
Удаляются устаревшие: `calib.whichDir` («Колесо %@ — куда крутилось?»), `calib.spinPrompt`, `calib.saveFailed`.

## Тестирование

- Компиляция `iphonesimulator26.2`.
- Симулятор (мок, `calibrated=false` → авто-открытие): пройти spin → tap → direction → … → done → save;
  скриншоты состояний; форс-харнесс для saving/failed. Машинка совпадает по размеру с экраном прошивки;
  кольца пульсируют только на spin/saving; failed — ✕ на колёсах. Обе темы.

## Вне объёма

- Прошивка/протокол `/calib*` (без изменений), диаграмма движения на главном.
- Вынесение пилюли/колец в общий компонент — допустимо локальное дублирование (рефактор позже).
