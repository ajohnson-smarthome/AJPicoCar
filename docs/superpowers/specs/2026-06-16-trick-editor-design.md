# Редактор трюков — время каждого действия

**Дата:** 2026-06-16
**Статус:** утверждён, готов к плану

## Проблема / идея

Сейчас на трюк есть один множитель длительности (`TrickSettings.scale`, экран «Трюки»
со слайдером на трюк). Но трюк = набор «действие → время». Хочется редактировать
**время каждого действия** трюка. Чтобы «виляние» (20 одинаковых чередующихся шагов)
не разворачивалось в 20 строк, «действие» = **различное движение** (уникальная `(t,y)`):
у каждого трюка 1–2 различных действия.

## Решение (iOS-only, прошивку не трогаем)

«Трюки»-список превращается в навигацию: тап по трюку открывает **редактор**, где
перечислены его различные движения, у каждого — слайдер времени (0.1–10 с) + сброс ↺.
Длительность хранится на трюк как массив (по одному значению на различное действие) в
`UserDefaults`. При проигрывании последовательность/число шагов берутся из базы
(`Tricks.swift`), а длительность каждого шага = отредактированное время его действия.
**Единичный множитель убираем** (его заменяют пер-действенные длительности).

## Модель и чистые хелперы (`Tricks.swift`, host-тест)

- **Действие** = различная пара `(t,y)` в шагах трюка, в порядке первого появления.
- `static func distinctActions(_ trick: Trick) -> [(t: Double, y: Double, count: Int)]` —
  уникальные `(t,y)` (первое появление) + сколько раз встречаются в трюке.
- `static func baseDurations(_ trick: Trick) -> [Int]` — базовая длительность каждого
  различного действия (ms первого шага этого действия; в наших трюках все шаги одного
  действия одинаковы по базе).
- `static func withDurations(_ trick: Trick, _ durs: [Int]) -> Trick` — пересобрать шаги:
  порядок/число шагов как в базе, у каждого шага `ms = clampDur(durs[indexOf its action])`.
  Если `durs` не той длины → вернуть базу. `id/nameKey/icon` сохраняются → `totalMs`
  отражает реальное время (кольцо прогресса остаётся верным).
- `static func clampDur(_ ms: Int) -> Int` — кламп в **[100, 10000]** (0.1–10 с).
- `static func actionDescriptor(_ t: Double, _ y: Double) -> (fwd: Int, turn: Int)` —
  знаки: `fwd ∈ {-1,0,1}` (вперёд/нет/назад по `t`), `turn ∈ {-1,0,1}` (влево/нет/вправо
  по `y`), порог `0.05`. Подпись действия собирается во View из этого дескриптора + L-токенов
  (без кириллицы в коде).

Старые scale-хелперы (`clampScale`/`scaled`/`scaledTrick`/`sliderToScale`/`scaleToSlider`)
удаляются; `DriveView` переходит на `withDurations`.

## Хранение (`TrickSettings.swift`)

- Ключ на трюк: `"trick.durs.\(id)"` → `[Int]` (мс на каждое различное действие, по порядку).
- `static func durations(for trick: Trick) -> [Int]` — из `UserDefaults` (если есть и длина
  совпадает с числом различных действий), иначе `Tricks.baseDurations(trick)`.
- `static func setDuration(_ trick: Trick, action i: Int, ms: Int)` — прочитать текущее,
  обновить `i`-й элемент (`clampDur`), записать массив.
- `static func reset(_ trick: Trick, action i: Int)` — вернуть `i`-й к базовому; если все
  стали базовыми — удалить ключ.
- Старый `"trick.scale.*"` больше не используется (безвредно остаётся в `UserDefaults`).

## UI

### «Трюки» (`TricksSettingsView`) — список-навигация
Строки больше не слайдеры: `NavigationLink` на `TrickEditorView(trick:)`, в строке —
иконка + название + суммарное время (`L.trickSec(totalSeconds)`) + шеврон. Заголовок/header
как сейчас.

### Редактор (`TrickEditorView.swift`)
- Header `‹ <название трюка>` (`dismiss`), система nav bar скрыта (как у под-экранов).
- Список различных действий трюка. Каждая строка:
  - **подпись действия** (из `actionDescriptor` + L-токены: «Вперёд», «Назад», «Поворот вправо/влево»,
    «Вперёд-вправо» и т.п.); под ней мелко «×N» если `count > 1`;
  - **слайдер** времени **0.1–10 с** (линейный), значение в секундах справа;
  - **↺** сброс этого действия к базе (тускл, когда уже база).
  - Запись в `TrickSettings` на отпускании слайдера; ↺ — сразу.
- Footer: «Всего: <totalSeconds> с · N движений · M шагов».

### Локализация
L-токены направлений + формат «×%d» + «Всего». Без кириллицы в коде; подпись действия
собирается во View функцией `actionLabel(descriptor) -> String` поверх `L`.

## Применение (`DriveView`)
`startTrick(base)`: `let durs = TrickSettings.durations(for: base);
let trick = Tricks.withDurations(base, durs)`; дальше как сейчас (стрим, кольцо, диаграмма).

## Host-тест (`TricksTests`)
- `distinctActions` (уникальность + count: spin 1, figure8 2, wiggle 2, donut 1);
- `baseDurations` (spin [5000], figure8 [2500,2500], wiggle [250,250], donut [5000]);
- `withDurations` (порядок/число шагов сохраняются, длительности применены к нужным шагам,
  `clampDur`, неверная длина → база);
- `actionDescriptor` (знаки по порогам).
Старые scale-тесты удаляются.

## Вне объёма
- Редактирование самого действия `(t,y)` и числа шагов (только время).
- Полный конструктор (добавить/удалить/переставить).
- Сброс всего трюка одной кнопкой (хватает ↺ на действие — их 1–2).
- Хранение на машинке / iCloud-синк.
