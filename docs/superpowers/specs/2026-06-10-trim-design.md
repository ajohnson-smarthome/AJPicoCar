# Триммер прямолинейности — Design

**Дата:** 2026-06-10
**Статус:** дизайн утверждён (обсуждение в чате), готов к плану

## Цель

Компенсировать разброс моторов (машинку уводит при езде прямо): один коэффициент `trim`,
который **ослабляет более быструю сторону** (никогда не усиливает — потолок 100%).
Настраивается из iOS-аппа, хранится в NVS.

## Семантика

`trim_pct` ∈ **−30..+30** (целые проценты; 0 = выкл):
- `trim_pct > 0` — ослабляем **левую** сторону (коррекция, когда уводит **вправо**);
- `trim_pct < 0` — ослабляем **правую** (уводит влево).

Чистая функция (применяется к скоростям сторон после `mixer_mix`, до `motors_plan`):
```c
// trim in [-0.3..0.3]; positive slows the LEFT side, negative slows the RIGHT.
static inline void trim_apply(float *left, float *right, float trim) {
    if (trim > 0) *left  *= (1.0f - trim);
    else if (trim < 0) *right *= (1.0f + trim);
}
```
Применяется и к отрицательным скоростям (реверс) — масштабирование симметрично знаку.
Спин/повороты тоже масштабируются — это ок (одинаковая компенсация физики стороны).

## Прошивка

- **`main/trim.h`** *(new)*: чистая `trim_apply` (zero IDF deps, `TRIM_HOST_TEST`-guard) —
  хост-тест `test/test_trim.c` (ослабление нужной стороны, знак/реверс, 0 = без изменений, кламп).
- **`main/car.c`**: `static float g_trim` (под `g_lock`, как `g_cfg`); в `car_drive` после
  `mixer_mix` → `trim_apply(&s.left, &s.right, g_trim)`; `car_set_trim(float)` / `car_get_trim()`
  (по паттерну `car_set_calibration` — checked take). Загрузка из NVS в `car_init`.
- **NVS**: ns `"car"`, ключ `"trim_pct"` (`nvs_get_i8`/`set_i8`), диапазон −30..30.
- **`main/trim_api.{c,h}`** *(new)*: `GET /trim` → `{"trim_pct":-5}`; `POST /trim` body `"-5"`
  (валидация −30..30) → `car_set_trim` + NVS save. Регистрация в `app_main`.

## iOS

- Пункт меню «Прямолинейность» (иконка `arrow.up.and.line.horizontal` или `alignhorizontal.center`)
  **между «Разгон» и «Прошивка»**.
- **`TrimView`** (split-layout): слева машинка-демо — рельсы-траектория **изгибаются в сторону
  коррекции** (trim>0 → влево), при 0 — прямые; колёса зелёные с шевронами (постоянная езда).
  Справа: заголовок «Прямолинейность» · подсказка **«Если машинку уводит — сдвинь ползунок в
  противоположную сторону.»** · слайдер **−30..+30, шаг 1, центр 0** · значение («Ровно» при 0,
  иначе «Влево/Вправо NN%»). POST на отпускании, GET в `.task`; демо обновляется на отпускании.
- **`TrimClient`** (как `RampClient`). Строки `trim.*` через `L`.
- **Мок**: `GET/POST /trim` (в памяти, дефолт 0).

## Строки

```
"trim.title"    = "Прямолинейность";
"trim.headline" = "Прямолинейность";
"trim.sub"      = "Если машинку уводит — сдвинь ползунок в противоположную сторону.";
"trim.center"   = "Ровно";
"trim.left"     = "Влево %d%%";
"trim.right"    = "Вправо %d%%";
```

## Тестирование

- Хост-тест `trim_apply`; `idf.py build`; сборка iOS; grep локализации.
- Симулятор: экран читает/пишет мок, демо изгибается по слайдеру, меню в порядке
  Калибровка / Разгон / Прямолинейность / Прошивка. Обе темы.
- Бенч (пользователь): пустить прямо → подобрать слайдером до ровной езды.

## Вне объёма

- Авто-калибровка увода (нет энкодеров), скоростезависимый трим, веб-пульт.
