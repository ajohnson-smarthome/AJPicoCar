# Анимированная графика экрана «Авто-возврат»

**Дата:** 2026-06-15
**Статус:** утверждён, готов к плану

## Проблема

`RecoverCarView` (графика слева на экране «Авто-возврат») статична, тогда как
другие экраны-настройки — `RampCarView` (демо разгона) и `TrimCarView` (крутящиеся
колёса + рельсы) — анимированы через `TimelineView(.animation)` с колёсами-чевронами.
Из-за этого «Авто-возврат» выбивается из общего стиля настроек.

## Решение

Привести `RecoverCarView` к тому же идиому: `TimelineView(.animation)`, колёса с
прокручивающимся протектором-чевроном (как Ramp/Trim), но в сторону **заднего хода**,
плюс пунктирный след позади, дашики которого «бегут» к машинке — машинка визуально
**отматывает путь назад**. Зависит от тоггла: ВКЛ → анимация идёт; ВЫКЛ → замирает
и гаснет.

## Детали
- **Обёртка:** `TimelineView(.animation) { tl in Canvas { ... } }` вместо статичного
  `Canvas`. Время `tl.date.timeIntervalSinceReferenceDate`.
- **Машинка:** геометрия/центрирование/размер **без изменений** (body 34×72, center =
  `size/2`, `frame(120×210).scaleEffect(1.6)`) — единый стиль со всеми экранами.
- **Колёса:** тёмная заливка + клиппинг + чевроны, прокрутка `phase = time × speed`
  (как `TrimCarView.drawWheel`), но направление **реверс** (протектор бежит назад →
  «задний ход»). Когда `active == false` — протектор не рисуется/застывший, колёса тёмные.
- **След:** пунктир позади (вниз от кормы) + реверс-шеврон на конце; дашики «маршируют»
  к машинке за счёт анимируемого `dashPhase` (`StrokeStyle.dashPhase`). При `active ==
  false` — статичный и приглушённый (как сейчас на off).
- **Скорость:** умеренная, в духе Ramp/Trim (`speed ≈ 60–70`); без рывков (непрерывная
  функция времени).

## Файлы
- **Изменяем:** `ios/ESP32Car/RecoverView.swift` — только `RecoverCarView` (тело →
  `TimelineView`+анимированные колёса/след). `RecoverView` (правая панель/логика) — без
  изменений; `active` уже прокидывается.

## Тестирование
- Сборка под симулятор.
- Галерея, кадр «Recover», обе темы: машинка анимирована (колёса/след движутся при ВКЛ,
  застывшие при ВЫКЛ), центрирование/размер совпадают с остальными экранами. Поскольку
  анимацию статичным скриншотом не зафиксировать однозначно — снять 2 кадра с интервалом
  и убедиться, что фаза протектора/дашей сместилась.

## Вне объёма
- Изменение геометрии машинки, правой панели, логики, REST/firmware.
- Цикл-демо «поехал вперёд → потерял связь → отмотал» (как фазовый цикл Ramp) — YAGNI;
  достаточно непрерывного реверс-протектора + бегущего следа.
