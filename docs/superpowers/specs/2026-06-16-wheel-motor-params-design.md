# Параметры колёс/моторов (база под измерение скорости)

**Дата:** 2026-06-16
**Статус:** утверждён, готов к плану

## Проблема / идея

Чтобы в будущем считать скорость машинки на борту (как спидометр: скорость колеса =
`π·D·обороты_вала/с`), нужно заранее знать **диаметр колеса** и **отсчётов на оборот
выходного вала (CPR)**. Энкодеров ещё нет — заложим **хранение этих параметров** в
прошивке + экран их редактирования сейчас, а сам расчёт скорости включим, когда
приедут моторы.

`CPR = PPR_энкодера × редуктор × квадратура`. Вводим PPR + редуктор + квадратуру,
приложение считает CPR. Плюс выпадающий список моделей моторов с автозаполнением.

## Объём СЕЙЧАС vs ПОТОМ

- **Сейчас (эта спека):** хранение параметров (D, PPR, редуктор, квадратура) в NVS +
  REST `/wheel`; iOS-экран «Колесо и моторы» (две карточки + пресеты моделей);
  экран как **шаг 1** обязательного мастера калибровки + **пункт меню** в настройках.
- **Потом (отдельная спека, когда приедут моторы):** чтение энкодеров через **PCNT** на
  C6 → `v = π·D·(тики/с)/CPR` → поле скорости в `/status`/телеметрии → индикатор в аппе.
  Параметры уже будут — останется «включить» расчёт.

## Прошивка

### `wheel.{c,h}` — хранилище параметров (NVS)
Параметры в NVS namespace `"car"`: `wheel_d` (u16, мм), `enc_ppr` (u16), `gear_x100`
(u16, редуктор×100 — 21.3 → 2130), `quad` (u8: 1/2/4). Геттеры/сеттеры; загрузка в
`wheel_init()` (дефолты ниже). Хелпер `wheel_cpr()` (= `ppr × gear × quad`, для будущего
расчёта скорости) — пока не используется, но заложен.
- **Дефолты при отсутствии в NVS:** D=65, PPR=11, gear_x100=2100 (1:21), quad=4.

### `wheel_api.{c,h}` — REST
`GET /wheel` → `{"diameter_mm":65,"ppr":11,"gear_x100":2100,"quad":4}`; `POST /wheel`
тем же JSON (валидация диапазонов), персист в NVS. Регистрация в `app_main` рядом с
прочими `*_api_start`. **Хендлеров станет 15/20** (`max_uri_handlers` не трогаем).

## iOS

### Данные пресетов — `MotorPresets.swift` (чистые, host-тест)
```swift
struct MotorPreset: Identifiable { let id: String; let name: String; let ppr: Int; let gear: Double; let quad: Int
    var cpr: Double { Double(ppr) * gear * Double(quad) } }
enum MotorPresets {
    static let all: [MotorPreset] = [
        // JGA25-370 (170 об/мин): 11 PPR · 1:21 · ×4 → CPR 924
        // JGB37-520B (1000 об/мин): 11 PPR · 1:9 · ×4 → CPR 396
    ]
    // helper: match(ppr,gear,quad) -> MotorPreset? (nil = свои параметры)
}
```
Стартовый список — только эти два мотора (числа проверяются по даташиту, поля редактируемые).
**Пункта «Другое» в меню нет** — поля PPR/редуктор/квадратура всегда редактируемые, и «Другое»
выбирать незачем. «Свои параметры» — это производная **подпись** строки «Модель», которая
показывается сама, когда текущие числа не совпадают ни с одним пресетом (`match` вернул `nil`).

### `WheelClient.swift`
`get() -> (diameterMm, ppr, gearX100, quad)?` и `set(...)` через `GET/POST /wheel` (JSON).

### `WheelParamsView.swift` — экран «Колесо и моторы»
Две секции (как в макете):
- **Колёса:** «Диаметр колеса» (`Stepper`, мм, шаг 1, ~20–150) + строка «Окружность πD».
- **Моторы:** «Модель» (**iOS `Menu`** popup → только реальные пресеты; выбор автозаполняет
  PPR/редуктор/квадратуру), «PPR» (`Stepper` ±1), «Редуктор» (числовое поле, десятичная
  клавиатура — дробный редуктор), «Квадратура» (segmented ×1/×2/×4) + строка вычисленного
  **CPR**.
- Правка любого из PPR/редуктор/квадратура → подпись строки «Модель» становится «Свои
  параметры» (`MotorPresets.match` вернул `nil`). Имя выбранной модели держим в `@AppStorage`
  для красивого ре-выбора; источник истины — числа (в прошивке).
- **Режим:** `var wizard: Bool = false`.
  - `wizard == false` (из меню): свой header «‹ Колесо и моторы» (`dismiss`), пишет в
    `WheelClient` на изменении.
  - `wizard == true` (шаг калибровки): заголовок «Параметры колёс» + «шаг 1 из 2»,
    кнопка **«Далее»** = `NavigationLink` на `CalibrationView(dismissible:false)`.
- Загрузка значений через `WheelClient.get()` в `.task`.

### Интеграция
- **Пункт меню** в `SettingsView`: «Колесо и моторы» (`systemImage` напр. `"ruler"` или
  `"gearshape.2"`) → `WheelParamsView()` (перед «Калибровка» или после — на ревью).
- **Мастер калибровки** (`DriveView` обязательный sheet): заменить
  `NavigationStack { CalibrationView(dismissible:false) }` на
  `NavigationStack { WheelParamsView(palette:p, wizard:true) }`; «Далее» пушит
  `CalibrationView(dismissible:false)`. Калибровка из настроек (`CalibrationView` напрямую)
  — без изменений.

### Локализация
Новые L-токены: заголовки/лейблы («Колесо и моторы», «Параметры колёс», «Диаметр колеса»,
«Окружность», «Модель», «Редуктор», «Квадратура», «Далее», «Свои параметры», «шаг %d из %d»),
форматы «%d мм»/«CPR %.0f». Без кириллицы в коде.

## Host-тест
Чистое в `MotorPresets`: `cpr` (ppr·gear·quad) и `match(ppr,gear,quad)` (находит пресет /
`nil` = свои параметры). `Tests` — отдельный `MotorPresetsTests`.

## Вне объёма
- Сам расчёт скорости, PCNT, поле скорости в `/status`/телеметрии (следующая спека, когда
  приедут моторы).
- Разный диаметр на колесо (один общий D).
- Хранение таблицы моделей на машинке (таблица — данные приложения).
