# Настройка трюков — длительность (множитель) на трюк

**Дата:** 2026-06-15
**Статус:** утверждён, готов к плану

## Проблема / идея

Сейчас трюки имеют фиксированные тайминги, зашитые в `Tricks.swift`. Нужна
возможность подстраивать длительность каждого трюка из приложения — экран
«Трюки» в настройках со списком всех трюков и слайдером на каждый. Длительность —
это, грубо, «скорость, с которой делается трюк» (дольше = больше оборот/размах,
короче = снапнее).

## Решение (всё в приложении, прошивку не трогаем)

База каждого трюка = **5 секунд** (при ×1). На трюк хранится **множитель**
длительности; при проигрывании длительности шагов умножаются на него. Множитель
настраивается логарифмическим слайдером **0.5×–12×** на экране «Трюки» в настройках;
у каждой строки — кнопка сброса к ×1. Хранится в `UserDefaults` телефона (трюки —
данные приложения; на машинке их нет).

## Компоненты

### Базовые тайминги (`Tricks.swift`) → 5 с на трюк
Каждый трюк при ×1 длится 5000 мс; шаги распределены так, чтобы сумма = 5000,
сохраняя характер трюка:
- **spin** `[ (0, 1, 5000) ]`
- **figure8** `[ (0.6, 0.6, 2500), (0.6, -0.6, 2500) ]`
- **wiggle** — снаппи: 20 шагов по 250 мс (10 циклов), генерируется
  `(0..<20).map { TrickStep(t: 0, y: $0 % 2 == 0 ? 0.8 : -0.8, ms: 250) }` (итого 5000)
- **donut** `[ (0.7, 1, 5000) ]`

Host-проверка `totalMs ≤ 20000` остаётся (база 5000 проходит).

### Хранение — `TrickSettings` поверх `UserDefaults`
- Ключ на трюк: `"trick.scale.\(id)"` → `Double`, дефолт **1.0**.
- `static func scale(_ id: Int) -> Double` (читает, дефолт 1.0, кламп [0.5, 12]).
- `static func setScale(_ id: Int, _ v: Double)` (кламп + запись).
- `static func reset(_ id: Int)` (= setScale 1.0).

### Чистая функция масштабирования (`Tricks`, host-тестируемо)
- `static let scaleMin = 0.5`, `scaleMax = 12.0`.
- `static func clampScale(_ v: Double) -> Double` — кламп в [0.5, 12].
- `static func scaled(_ steps: [TrickStep], by scale: Double) -> [TrickStep]` —
  каждый `ms → max(1, Int((Double(ms) * clampScale(scale)).rounded()))`; `t,y` не трогаются.
- Логарифмический маппинг слайдера ↔ множитель (для UI), тоже чистый:
  - `sliderToScale(_ p: Double) -> Double = scaleMin * pow(scaleMax/scaleMin, p)` (p∈[0,1])
  - `scaleToSlider(_ s: Double) -> Double = log(s/scaleMin) / log(scaleMax/scaleMin)`
  - Равные движения слайдера = равное изменение «во сколько раз». ×1 садится на
    ~22% хода (геометрически диапазон 0.5–12 несимметричен относительно 1 — это ок,
    подпись в секундах снимает неоднозначность).

### Применение при проигрывании (`DriveView`)
`startTrick(trick)`: `let s = TrickSettings.scale(trick.id)`, шаги =
`Tricks.scaled(trick.steps, by: s)`, стримит их. Кольцо прогресса: длительность =
сумма масштабированных `ms` (реальное время трюка). База в палитре трюков не меняется.

### Экран «Трюки» (`TricksSettingsView`)
- `NavigationLink` в `SettingsView` (после «Авто-возврат», до «Прошивка»),
  `Label(L.tricksTitle, systemImage: "sparkles")`.
- **Список** (как `SettingsView`, НЕ `SplitScreen`): свой header «‹ Трюки» + `dismiss()`,
  `.toolbar(.hidden, for: .navigationBar)` (консистентно с другими под-экранами).
- Каждая из 4 строк: иконка трюка + название + **лог-слайдер** (0…1, маппится в множитель) +
  значение справа (секунды `%.1f с` крупно + `×%.1f` мелко) + кнопка **↺** сброса к ×1
  (подсвечена при ≠1, тусклая/неактивная при =1).
- Слайдер пишет в `TrickSettings` на отпускании (как ramp/trim); ↺ — сразу.

### Локализация
`tricks.title` = «Трюки» (пункт меню + заголовок экрана), `tricks.sec` = «%.1f с»,
`tricks.mult` = «×%.1f». Без кириллицы в коде (суффикс «с» — из strings).

## Host-тест
В `TricksTests`: `clampScale` (границы), `scaled(by:)` (`ms ≥ 1`, `t,y` неизменны,
масштаб применён), `sliderToScale`/`scaleToSlider` (обратимость, концы диапазона,
×1 в середине лог-шкалы по значению).

## Вне объёма
- Регулировка скорости (t,y), пер-шаговое редактирование.
- Глобальный сброс «всё к ×1» (footgun — только пер-трюк ↺).
- Синхронизация через iCloud, хранение на машинке.
- «Превью» трюка из настроек (нужны моторы).
