# Трюки (макросы манёвров) — iOS-only

**Дата:** 2026-06-15
**Статус:** утверждён, готов к плану

## Проблема / идея

Дать машинке набор «трюков» — одной кнопкой запускать заранее заданные манёвры
(разворот, восьмёрка, вилять, пончик). Для веселья/демо. Датчиков нет → манёвры
**open-loop** (таймированные, без точной геометрии — угол зависит от поверхности/заряда).

## Решение (архитектура B — всё в приложении)

Трюк = таймлайн шагов `{t, y, длительность}`. Приложение **стримит** эти команды
через существующий `CarConnection.setCommand` (10 Гц-таймер уже шлёт текущую команду
на `/ws`), затем останавливает. **Прошивка не меняется** — трюк выглядит для машинки
как обычная езда по скрипту. Касание джойстика в любой момент бросает скрипт и
возвращает ручное управление.

Почему B, а не firmware-проигрыватель: ноль изменений прошивки (на плате уже
`v1.0+311`), макросы правятся в Swift без перепрошивки, прерывание тривиально
(отменяемая `Task`), джиттер на локальном softAP при 10 Гц некритичен.

## Компоненты (iOS)

### `Tricks.swift` (чистые данные, host-проверяемо)
```swift
struct TrickStep { let t: Double; let y: Double; let ms: Int }
struct Trick: Identifiable {
    let id: Int            // 1..4
    let nameKey: String    // → L
    let icon: String       // SF Symbol
    let steps: [TrickStep]
    var totalMs: Int { steps.reduce(0) { $0 + $1.ms } }
}
enum Tricks {
    static let all: [Trick] = [ spin, figure8, wiggle, donut ]
    // spin    ↻ arrow.clockwise        — [ (0, 1, 1200) ]
    // figure8 ∞ infinity               — [ (0.6, 0.6, 1400), (0.6, -0.6, 1400) ]
    // wiggle  〰 wind                    — [ (0,0.8,250),(0,-0.8,250) ]×3
    // donut   ◯ circle.dashed          — [ (0.7, 1, 2000) ]
}
```
Все `t,y ∈ [-1,1]`, `ms > 0` — проверяется host-тестом (swiftc, как `ControlModel`).
Тайминги — стартовые; подкрутим на железе позже (правка чисел в Swift, без прошивки).

### Проигрыватель (в `DriveView`)
Состояние:
- `@State private var runningTrick: Trick?`
- `@State private var trickTask: Task<Void, Never>?`

Запуск `startTrick(_ trick)`:
1. отменить текущий `trickTask`;
2. `runningTrick = trick`;
3. `trickTask = Task { for step in trick.steps { conn.setCommand(frame(step.t, step.y)); try? await Task.sleep(ms: step.ms); if Task.isCancelled { return } }; conn.setCommand(frame(0,0)); runningTrick = nil }`.

Существующий 10 Гц-таймер шлёт текущую команду на машинку; шаг длительностью N мс
= таймер отправит её ~N/100 раз. По завершении — `(0,0)` (стоп) и сброс состояния.

Отмена `cancelTrick(stop: Bool)`:
- `trickTask?.cancel(); trickTask = nil; runningTrick = nil`;
- если `stop == true` → `conn.setCommand(frame(0,0))` (явный стоп через ⏹);
- если `stop == false` → команду НЕ трогаем (джойстик сам её выставит — бесшовный перехват).

**Перехват джойстиком:** в начале `push()` (вызывается на любое ручное касание/гейм-пад):
`if runningTrick != nil { cancelTrick(stop: false) }` — затем обычная отправка команды
стика. То есть стик всегда главнее трюка, прерывание без стопа (как abort в recovery).

### UI: `TricksControl.swift` (FAB ✦ + поповер C4)
Презентационный компонент в нижне-центральной зоне `DriveView` (между джойстиками):
- **FAB** Ø~46, по центру снизу, чуть выше статус-строки. Состояния:
  - покой → `✦` (тап: открыть/закрыть поповер);
  - открыт → `✕` (тап: закрыть);
  - `running != nil` → `⏹` (тап: `onStop`).
- **Поповер C4** (при открытии, скрыт когда `running`): карточка над FAB с «хвостиком»,
  4 строки `[иконка + название]`; тап строки → `onSelect(trick)` + закрыть.
- Интерфейс: `TricksControl(palette:, running: Trick?, onSelect: (Trick)->Void, onStop: ()->Void)`;
  `open` — внутреннее состояние. `DriveView` владеет `runningTrick`/`trickTask`.
- Индикация хода: пока `running != nil` — короткая подпись «Трюк: <название>» в статус-строке
  (необязательно; FAB=⏹ уже сигнализирует).

### Локализация
Имена трюков и подпись — через `enum L` + `Resources/ru.lproj/Localizable.strings`
(`tricks.spin/figure8/wiggle/donut`, `tricks.running`). Без кириллицы в коде.

### Галерея
В `GalleryView` существующие кадры «Drive arcade/tank» уже показывают `DriveView` —
FAB будет виден там; отдельный кадр не нужен (можно добавить «Drive tricks» с открытым
поповером через `preview`-сид, если удобно для скриншота).

## Тестирование
- `swiftc` host-проверка `Tricks.all`: каждый трюк непустой, `t,y ∈ [-1,1]`, `ms > 0`,
  `totalMs` ограничен (≤ ~6 с) — как существующие чистые Swift-проверки.
- Сборка под симулятор; галерея — FAB/поповер C4 в обеих темах, центрирование между
  джойстиками. Само выполнение трюка (движение) — на железе (моторы), как и recovery.

## Безопасность / поведение
- Трюк прерывается: **касанием джойстика** (перехват без стопа) или **⏹** (стоп). Иначе
  доигрывает и встаёт `(0,0)`.
- Машинка под watchdog как обычно (кадры идут 10 Гц во время трюка → не сработает; после
  завершения, в покое, поведение штатное).
- Трюк-кадры попадают в breadcrumb recovery (как любая езда) — безвредно.

## Вне объёма
- Изменения прошивки / новый REST (схема B их не требует).
- Слайдеры подстройки таймингов каждого трюка (правим числа в `Tricks.swift`).
- Запись пользовательских макросов; точные углы (нужен гироскоп).
- Жёсткая блокировка джойстиков на время трюка (вместо неё — перехват касанием).
