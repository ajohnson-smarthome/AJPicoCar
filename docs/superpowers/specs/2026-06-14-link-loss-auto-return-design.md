# Авто-возврат при потере связи (breadcrumb-откат)

**Дата:** 2026-06-14
**Статус:** утверждён, готов к плану

## Проблема

Машинка — это softAP; телефон подключается к ней. Когда машинка уезжает за
пределы дальности своего AP, WS-кадры перестают доходить, watchdog (>300 мс без
кадра) зовёт `car_stop()`, и машинка встаёт на краю зоны. Дальше она **не может
вернуться сама** — команд нет, а датчиков положения/одометрии на борту нет.

## Решение

Вести кольцевой буфер последних N секунд команд (каждый WS-кадр = сэмпл
`{throttle, yaw, timestamp}`) и при обрыве **проигрывать его в обратном порядке,
каждый сэмпл реверснутым**, пока не вернётся связь. Это отматывает реальную
пройденную траекторию по заведомо свободной земле. Длина окна настраивается
ползунком (1–10 с) в iOS-настройках; фича включается/выключается тогглом.

Почему реверс сэмпла = ретрейс: для дифференциального привода путь задаётся
скоростями сторон; отрицание `(throttle, yaw)` отрицает обе скорости сторон
(`left=t+y`, `right=t−y`), что ведёт машинку назад по той же дуге. Проигрывание
сегментов в обратном временном порядке ретрейсит всю траекторию.

## Архитектура

### Новый модуль `main/recovery.{c,h}`

Watchdog перестаёт звать `car_stop()` напрямую — теперь зовёт
`recovery_on_link_lost()`, и модуль решает: просто стоп или откат. Вся логика
возврата в одном месте; watchdog остаётся «тупым».

**Состояние (в `recovery.c`):**
- `enabled` (bool, из NVS) и `window_ms` (uint16, из NVS).
- Кольцевой буфер сэмплов `{float t; float y; uint32_t ts_ms;}`.
- `seq` (uint32, volatile) — инкрементится на каждый принятый кадр; признак
  «связь жива» для прерывания отката.
- Задача-ретрит (FreeRTOS task), спит на нотификации.

**Размер буфера выведен из окна (статическая аллокация):**
```
#define FRAME_HZ        10                       // частота WS-стрима (телефон шлёт 10 Гц)
#define WINDOW_MAX_S    10                        // максимум ползунка
#define MAX_SAMPLES     (WINDOW_MAX_S * FRAME_HZ * 3 / 2)   // =150, запас ×1.5 на джиттер
```
Рантайм-окно (`window_ms`, 1000–10000) управляет вытеснением: сэмплы старше окна
от «сейчас» выбрасываются; replay идёт только по сэмплам внутри окна.

**API модуля:**
- `void recovery_init(void)` — грузит `enabled`/`window_ms` из NVS (дефолты ниже),
  создаёт задачу-ретрит. Вызвать ДО `watchdog_init()`.
- `void recovery_note_command(float t, float y)` — из `ws_control` на каждый кадр:
  пишет сэмпл `{t, y, now_ms()}` в буфер, вытесняет старьё за окном, `seq++`.
- `void recovery_on_link_lost(void)` — из watchdog при трипе. Если выключено,
  буфер пуст или весь путь в окне ≈ стоп → `car_stop()`. Иначе будит задачу-ретрит.
- `void recovery_set_config(bool enabled, uint16_t window_ms)` /
  `void recovery_get_config(bool *enabled, uint16_t *window_ms)` — для API-слоя
  (RAM; персист в NVS делает `recovery_api`, как у `trim`).

**Задача-ретрит (проигрывание истории назад):**
1. Снять снимок `seq`; зафиксировать момент обрыва `t_loss = now`.
2. Идти по сэмплам внутри окна **новейший → старейший**:
   - длительность сегмента `i` = `ts_{i+1} − ts_i`; для самого новейшего —
     `min(t_loss − ts_last, WDT_TIMEOUT)` (он держался до трипа watchdog);
   - команда сегмента: `car_drive(-t_i, -y_i)`;
   - выждать длительность сегмента **тиками ~30 мс**; на каждом тике: если `seq`
     изменился → **abort** (пришёл новый кадр, связь вернулась; выход без стопа —
     новая команда уже владеет ramp).
3. Дошли до конца окна (связь не вернулась) → `car_stop()`.

Гарантии: слепой ход ≤ окна (≤10 с), по заведомо свободной только что пройденной
земле; реагирует на возврат связи в пределах ~30 мс; всегда заканчивается стопом.
**Один откат на один обрыв** (watchdog после трипа дизармится, ре-армится только
при возврате трафика — а это уже abort).

### Интеграция в существующий код
- `main/watchdog.c`: в `wdt_cb` заменить `car_stop()` на `recovery_on_link_lost()`.
  (Счётчик трипов и дизарм — без изменений.)
- `main/ws_control.c`: в обработчике кадра рядом с `watchdog_feed()` добавить
  `recovery_note_command(t, y)`.
- `main/main.c`: `recovery_init()` непосредственно перед `watchdog_init()`;
  `recovery_api_start()` рядом с `trim_api_start()`.
- Console-путь не трогается (он и так вне watchdog → откат его не касается).

### REST `main/recovery_api.{c,h}`
- `GET /recover` → `{"enabled":bool,"window_ms":N}`.
- `POST /recover` тело JSON `{"enabled":..,"window_ms":..}` (два поля → JSON, а не
  plain-число как `/trim`; валидация: `window_ms` ∈ [1000,10000]) → `recovery_set_config`
  + персист в NVS namespace `"car"`: `recover_en` (i8 0/1), `recover_win` (u16, мс).
- Регистрация в `app_main` рядом с прочими `*_api_start`. Хендлеров станет 13
  (лимит `max_uri_handlers = 20` — запас есть).

### NVS / дефолты
- `recover_en` = **1 (ВКЛ)**, `recover_win` = **5000 (5 с)** при отсутствии в NVS.
- `recovery_init` читает их из `"car"` (как `car_init` читает `trim_pct`).

### iOS
- `RecoverClient.swift` — `get() -> (enabled: Bool, windowMs: Int)?` и
  `set(enabled:windowMs:) -> Bool` через `GET/POST /recover` (JSON, как `/status`).
- **Экран «Авто-возврат»** (`RecoverView.swift`, на `SplitScreen` как `RampView`):
  справа — `Toggle` «Авто-возврат при обрыве» + `Slider` «Окно: N с» (1–10 с,
  неактивен когда выкл) + подпись-пояснение; слева — `RecoverCarView`.
  `NavigationLink` в `SettingsView` (после «Прямолинейность»).
- **`RecoverCarView`** обязан использовать **ту же геометрию машинки, что
  `RampCarView`/`TrimCarView`**: `Canvas` с `center = size/2`, body 34×72, тёмные
  угловые колёса 11×15, лобовое; обёртка `.frame(width:120, height:210).scaleEffect(1.6)`.
  Машинка остаётся **по центру левой половины, того же размера, что на всех экранах**
  (единый стиль из правки SplitScreen) — её НЕ сдвигать. Пунктирный «след» (траектория
  отката) рисуется в том же `Canvas` **позади/ниже** машинки от её кормы (декорация,
  центр машинки не двигает); неактивен/погашен, когда тоггл выключен.
- Строки — через `enum L` + `ru.lproj` (без кириллицы в коде).

## Host-тесты
Чистые предикаты в `recovery` (компилируются `cc` без ESP-IDF):
- `recovery_reverse(float t, float y, float *rt, float *ry)` — отрицание.
- `recovery_evict(uint32_t ts, uint32_t now, uint16_t window_ms)` → bool — сэмпл
  старше окна.
→ `test/test_recovery.c`, добавить в `test/Makefile` (`make run`).
Кольцевой буфер и тайминг-цикл — RTOS, на хосте не тестируются.

## Вне объёма
- Коалесинг одинаковых подряд команд (с аналоговым стиком почти не помогает).
- Счётчик возвратов в телеметрии/`/status` (можно добавить позже).
- Анимированное демо на экране «Авто-возврат» (хватит статичного графа).
- Настройка частоты кадров (`FRAME_HZ` — компайл-тайм константа = стрим 10 Гц).
