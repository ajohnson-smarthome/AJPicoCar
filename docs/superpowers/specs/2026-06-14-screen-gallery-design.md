# Галерея экранов (debug-only превью) — Design

**Дата:** 2026-06-14
**Статус:** дизайн утверждён (обсуждение в чате), готов к плану

## Цель

Инструмент разработчика: листать **все экраны приложения во всех состояниях** по очереди тапом, с
подставным (mock) состоянием — для визуального аудита и демо/скриншотов без реального железа.

## Доступ

- **Только `#if DEBUG`** — в релизной сборке кода галереи нет вообще.
- Включается **launch-аргументом** `-gallery` (`ProcessInfo.processInfo.arguments.contains("-gallery")`),
  чтобы обычная debug-разработка не «захватывалась» галереей. Запуск:
  `xcrun simctl launch booted <id> --args -gallery` (или аргумент в Xcode-схеме).
- В `ESP32CarApp`: `#if DEBUG` и аргумент есть → корень = `GalleryView`; иначе обычный `AppFlow`-флоу.

## Навигация

- Тап по **левой** половине экрана → предыдущий кадр; по **правой** → следующий. Зацикливание по кругу.
- **Плашка-счётчик** сверху (поверх кадра): «6 / 26 · Firmware (downloading)».

## Архитектура

- **`GalleryView.swift`** (`#if DEBUG`): `frames: [(label: String, view: AnyView)]` + `@State index` +
  ZStack с двумя невидимыми tap-зонами (лево/право) + overlay-счётчик. Палитра из `Theme.current`.
- **Подставное состояние** (`#if DEBUG`, в `GalleryView` или соседнем `GalleryMocks.swift`):
  - `mockStatus(online:calibrated:fw:rssi:wdtTrips:) -> CarStatus` — создаёт `CarStatus` и проставляет
    `@Published`-поля напрямую (online/calibrated/fw/rssi/wdtTrips/uptimeS) для нужного кадра.
  - мок `CarConnection` (не подключается; DriveView рисует джойстики/диаграмму из локального стейта),
    мок `UpdateClient` (downloadProgress задаётся для кадра «downloading»).
  - Экраны-чистые-функции (Connect, NoInternet, Drive, Ramp, Trim, Settings) строятся прямо с моками.
- **Debug-хуки в экранах со внутренней стейт-машиной** (под `#if DEBUG`):
  - `FirmwareView` — опц. `debugPhase: FwPhase? = nil`: если задан, `.task`-проверка (`check()`)
    пропускается и `phase` инициализируется этим значением (показ статичной фазы). Форс-кадр —
    `forced: true` + `debugPhase: .available`.
  - `CalibrationView` — опц. `debugState` (тип под её внутренний CalState/шаг): если задан, рисует
    это состояние без сетевых вызовов.
  Оба параметра существуют только в `#if DEBUG` и по умолчанию nil → прод-поведение не меняется.

## Тестирование

- Чистого алгоритма нет (визуальный инструмент). Проверка:
  - сборка `iphonesimulator26.2`; grep кириллицы (тексты галереи-счётчика — латиницей/числами, без новых
    локализуемых строк) и цветов.
  - запуск в симуляторе с `--args -gallery` → скриншоты первого кадра и пары промотанных; тап лево/право
    переключает; счётчик корректен.
  - запуск **без** `-gallery` → обычный флоу (галерея не активна), drive как прежде.

## Вне объёма

- Прод-доступ (галерея строго debug-only).
- Анимации переходов между кадрами (простая смена), сохранение/экспорт скриншотов.
- Локализация подписей счётчика (отладочный текст, латиница — не через `L`).
