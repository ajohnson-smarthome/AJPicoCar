# iOS: мастер калибровки — Design

**Дата:** 2026-06-08
**Статус:** дизайн утверждён, готов к плану

## Цель

Реальный мастер калибровки в iOS-приложении (вместо заглушки): сопоставить 4 мотор-пары физическим
колёсам и их направлению, как в веб-пульте, и сохранить в машинку через REST `/calib*`. Тестируется
в симуляторе против mock-сервера (поток UI), реально калибрует на устройстве.

## Протокол прошивки (уже существует, не трогаем)

- `GET /calib` → `{"calibrated":true|false}`.
- `POST /calib/spin` body `"<pair>,<dir>"` — pair 0–3, dir **1=вперёд / 0=назад**; пульсирует пару ~0.6 с.
- `POST /calib/save` body `"<p>:<s>,<p>:<s>,<p>:<s>,<p>:<s>"` — для **FL,FR,RL,RR**; p=индекс пары (0–3),
  s=знак (±1). Ответ `ok` или 400.

## Поток мастера (как в вебе)

`CalibrationView` — диаграмма машинки сверху (та же эстетика, что `DriveDiagram`) с 4 тапаемыми
колёсами (FL/FR/RL/RR). Шаги для `i` = 0…3:
1. Кнопка **Spin** → `POST /calib/spin "i,1"` (крутит пару `i` вперёд ~0.6 с).
2. Пользователь смотрит, какое колесо крутанулось, **тапает его** на диаграмме.
3. Выбирает **направление** (вперёд/назад) → назначение `угол ← (пара i, знак +1/−1)`; колесо
   помечается ✓ и больше не тапается; шаг++.
4. После 4 назначений → кнопка **Save** активна → `POST /calib/save` со строкой в порядке
   FL,FR,RL,RR → `ok` → калибровано, экран закрывается. При ошибке — сообщение, можно повторить.

## Вход

- **Шестерёнка → Настройки → «Калибровка»** открывает `CalibrationView` (заменяет стаб).
- **Авто:** если машинка не калибрована (`status.calibrated == false` из `/status`) и онлайн —
  при подключении один раз предложить калибровку (презентовать `CalibrationView` листом). После
  закрытия в этой сессии повторно не всплывает (флаг `didPromptCalib`).

### Модель данных
- `enum Corner: CaseIterable { case fl, fr, rl, rr }` (порядок = FL,FR,RL,RR).
- Назначение: `[Corner: (pair: Int, sign: Int)]`.
- `ControlModel.calibSaveBody(_ a: [Corner:(pair:Int,sign:Int)]) -> String` → `"p:s,p:s,p:s,p:s"`
  в порядке `Corner.allCases`. Предусловие: все 4 угла заданы (мастер не зовёт раньше).

## Тестирование

- **Хост-тест (нативно/симулятор):** `calibSaveBody` — порядок FL,FR,RL,RR и формат `p:s`
  (например FL=(0,+1),FR=(1,-1),RL=(2,+1),RR=(3,-1) → `"0:1,1:-1,2:1,3:-1"`).
- **Мок:** `curl http://127.0.0.1:8080/calib` → `{"calibrated":false}`; после `POST /calib/save`
  → `true`; `/calib/spin` логирует пару.
- **Симулятор:** старт → `status.calibrated=false` → авто-лист калибровки; пройти 4 шага
  (Spin/тап/направление) → Save → лист закрывается, статус-строка `calib ✓`; повторно через ⚙.
  Обе темы.
- **Устройство:** реальная калибровка (моторы крутятся, идентификация по физическому колесу).

## Вне объёма

- Изменение прошивки (эндпоинты есть).
- Хранение/отображение текущей калибровки в приложении (только запись).
- Частичная/одно-колёсная перекалибровка — только полный проход 4 пар.
