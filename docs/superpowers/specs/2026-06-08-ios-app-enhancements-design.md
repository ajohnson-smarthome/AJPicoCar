# iOS-приложение: апгрейд UI + телеметрия + `/status` — Design

**Дата:** 2026-06-08
**Статус:** дизайн утверждён (через визуальный компаньон), готов к плану реализации

## Цель

Развить нативный пульт (Фаза 1): сделать интерфейс красивым и лаконичным с поддержкой
**светлой и тёмной тем**, добавить **точную проверку «мы на сети машинки»** (через новый
firmware-эндпоинт `/status`), и расширить экран **телеметрией** (пинг, скорость/поворот,
анимация колёс, статус машинки). Перестилизовать переключатель схем (плохо виден на тёмной).

## Решения (выбраны в визуальном компаньоне)

| Аспект | Решение |
|---|---|
| Раскладка | **B1**: верх — связь+пинг (слева) · переключатель (справа); центр — `L · машинка · R` на одной оси; низ по центру — мелкая инфа (uptime/calib/fw); джойстики в нижних углах |
| Темы | **обе тёплые**, приложение следует системной теме iOS. Тёплая тёмная (`#110f0c`, тёплые серо-бежевые тексты); тёплая светлая (`#f4f0e8`, кремовые панели). Акцент: зелёный `#4ade80` (тёмная) / `#15803d` (светлая) |
| Проверка сети | новый firmware `GET /status` с подписью `device:"esp32-car"`; приложение опрашивает — нет/чужой ответ → экран «перейди на Wi-Fi ESP32-Car» + кнопка открыть Настройки |
| Телеметрия | пинг (RTT, реальный), скорость/поворот (L/R % — вычисленная команда), анимация колёс (стиль **B: стрелка + свечение**), статус из `/status` |
| Переключатель | кастомный сегмент-контрол (не дефолтный `Picker`) под темы, с акцентом — хорошо виден |

**Честно об источниках данных:** обороты колёс прошивка НЕ меряет (нет энкодеров). Скорость/
поворот/анимация — это **команда** (вычисленные `t,y` → борта), а не реальные RPM. Пинг и поля
`/status` — реальные.

## Часть 1 — Прошивка: эндпоинт `/status` (маленькое добавление)

Новый `main/status_api.{c,h}` (или расширить `calib_api`): регистрирует `GET /status` на общем
httpd (`http_server_get_handle()`), возвращает JSON:
```json
{"device":"esp32-car","fw":"1.0","uptime_s":1234,"calibrated":true,"heap":210000}
```
- `device` — фиксированная подпись (позитивная идентификация машинки).
- `uptime_s` — `esp_timer_get_time()/1000000`.
- `calibrated` — `calibration_load()` проба.
- `heap` — `esp_get_free_heap_size()`.
- `fw` — `#define`.
Вызвать `status_api_start()` в `app_main` после `http_server_start()`. Аддитивно, ничего не ломает.
(Замечание: captive-портал убран ранее — `/status` к этому отношения не имеет.)

## Часть 2 — iOS: темы

`Theme`/`Palette` — набор цветов, выбираемый по `@Environment(\.colorScheme)`:
- **dark (тёплая):** bg `#110f0c`, panel `#1b1711`, line `#2b251c`, text `#b3ac9f`, muted `#6b6356`, accent `#4ade80`, idleWheel `#3a5a44`.
- **light (тёплая):** bg `#f4f0e8`, panel `#fffdf8`, line `#e8dfcf`, text `#6a635a`, muted `#a59c8e`, accent `#15803d`, idleWheel `#cfe3d2`.
Все вью читают цвета из текущей палитры. Приложение **следует системной теме** (без ручного
тумблера в Фазе 1; можно добавить позже).

## Часть 3 — iOS: проверка сети машинки

`CarConnection` (или новый `CarStatus`) при старте и периодически (например раз в 2 с, пока не
на машинке) делает `GET http://192.168.4.1/status` с коротким таймаутом:
- 200 + `device=="esp32-car"` → **на машинке** → показываем пульт, открываем WS `/ws`.
- таймаут/ошибка/другой `device` → **не на той сети** → `ConnectView`: «Подключись к Wi-Fi
  **ESP32-Car** (пароль `drive1234`)» + кнопка **открыть Настройки** (`UIApplication.open`
  системного URL — прямого диплинка в Wi-Fi-панель нет). Авто-повтор опроса.
Это заменяет прежнюю ping-проверку (позитивная идентификация надёжнее).

## Часть 4 — iOS: телеметрия на экране (раскладка B1)

- **Верхняя панель:** слева `● connected · <ping> ms` (точка = статус связи; ping = RTT
  последнего `/status`/ws-ping); справа — кастомный переключатель **Arcade/Tank**.
- **Центр — `L · машинка · R`:** L/R % = скорости бортов из текущей команды
  (`left=t+y, right=t-y`, нормализовано, в %). По центру — диаграмма машинки сверху с
  **анимацией колёс (стиль B)**: каждое колесо — стрелка `↑`/`↓` (направление борта) и
  яркость/пульс свечения по `|скорость|`. Левые колёса = левый борт, правые = правый
  (skid steer; знак из калибровки в приложении не отражаем — показываем по бортам).
- **Низ по центру:** мелкая инфа из `/status`: `up <uptime> · calib ✓/✗ · fw <ver>`.
- **Джойстики** — в нижних углах (как в Фазе 1): аркада (правый стик; в этой раскладке шкалу
  газа заменяет анимация колёс) / танк (два стика).

## Архитектура iOS (файлы)

| Файл | Изменение |
|---|---|
| `Theme.swift` *(new)* | палитры dark/light + аксессор по colorScheme |
| `CarStatus.swift` *(new)* или расширение `CarConnection` | опрос `/status`, парсинг, ping, поля статуса (`ObservableObject`) |
| `DriveView.swift` | новая раскладка B1; верхняя панель; центр L·car·R; низ-статус; темы |
| `WheelsView.swift` *(new)* | диаграмма машинки + анимация колёс (стрелка+свечение) из команды |
| `SchemeToggle.swift` *(new)* | кастомный сегмент-контрол под темы |
| `ConnectView.swift` | под темы + текст «перейди на ESP32-Car» + кнопка Настройки (есть) |
| `ControlModel.swift` | + хелпер `sides(t,y) → (left,right)` (борта в %) для телеметрии, **юнит-тест** |

## Тестирование

- **Юнит-тест (нативно/симулятор):** `ControlModel.sides(t,y)` — борта/нормализация (зеркалит миксер).
- **Сборка:** `xcodebuild` под симулятор-SDK; темы — превью обеих в Xcode/на устройстве.
- **Прошивка:** `idf.py build`; `curl http://192.168.4.1/status` после подключения к сети.
- **На устройстве:** обе темы (переключить системную тему iOS); вне сети машинки → экран
  «перейди на ESP32-Car»; на сети → пульт + живая телеметрия (ping, L/R %, колёса, статус).

## Открытые параметры (не блокируют)

- Период опроса `/status` (старт 2 с офлайн / 1 с онлайн для ping).
- Ручной тумблер темы (отложено — пока следуем системной).
- Точные тайминги пульсации свечения колёс (подбор на устройстве).

## Вне объёма

- Реальные обороты колёс (нужны энкодеры/датчики тока — железо).
- Авто-подключение к Wi-Fi (платный entitlement) и чтение SSID (платный entitlement) — обходим
  через `/status`-идентификацию.
- Калибровка в приложении — отдельная Фаза 2 (`docs/.../2026-06-08-ios-app-design.md`).
