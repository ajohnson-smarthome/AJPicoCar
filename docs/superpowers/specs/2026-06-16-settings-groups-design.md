# Группировка пунктов меню «Настройки»

**Дата:** 2026-06-16
**Статус:** утверждён, готов к плану

## Проблема

`SettingsView` показывает 7 пунктов одним плоским списком (Колесо и моторы, Калибровка,
Разгон, Прямолинейность, Авто-возврат, Трюки, Прошивка). По мере роста списка стало
непонятно, что к чему относится. Нужно сгруппировать по логическому смыслу с заголовками
секций.

## Решение (вариант A, утверждён)

Разбить список на **3 секции** (нативный SwiftUI `Section` внутри текущего `List`):

| Секция (заголовок) | Пункты (в этом порядке) |
|---|---|
| **Настройка машины** | Колесо и моторы · Калибровка |
| **Движение** | Разгон · Прямолинейность · Авто-возврат · Трюки |
| **Система** | Прошивка |

Логика: сверху — разовая настройка железа (геометрия колёс/моторов + калибровка
направления вращения); в середине — как машина едет (разгон, прямолинейность, авто-возврат
при обрыве, трюки); снизу — система (обновление прошивки).

Плюс: **сменить иконку** «Колесо и моторы» с `ruler` на **`steeringwheel`** (тонкий
монохромный SF Symbol, рифмуется с настройкой колёс, не дублирует шестерёнки «Калибровки»).

## Объём

Чисто презентационное изменение **одного экрана**. Прошивки, сетевого слоя, логики
навигации не касается. Затрагивает:
- `ios/ESP32Car/SettingsView.swift` — обернуть 7 существующих `NavigationLink` в три `Section`
  с заголовками; сменить `systemImage` первого пункта на `steeringwheel`.
- `ios/ESP32Car/L.swift` — 3 новых аксессора заголовков секций.
- `ios/ESP32Car/Resources/ru.lproj/Localizable.strings` — 3 новые строки.

## Детали оформления

- Заголовки секций — в стиле приложения: приглушённый цвет (`palette.muted`), мелкий
  размер (~11–12 pt). Если стандартный `Section`-header в текущем `List` (с
  `.scrollContentBackground(.hidden)` + `.listRowBackground(palette.panel)`) рендерится
  системным капсом/отступом, задать `Text(...).foregroundStyle(palette.muted)` и при
  необходимости `.listRowInsets`/`.textCase(nil)` чтобы выглядело ровно.
- У каждой строки **сохраняется** `.listRowBackground(palette.panel)` и
  `.foregroundStyle(palette.text)` — как сейчас.
- Порядок пунктов внутри секций — как в таблице выше. Существующие иконки остальных
  пунктов не меняются (`gearshape.2`, `gauge.with.needle`, `arrow.up.to.line`,
  `arrow.uturn.backward`, `sparkles`, `arrow.down.circle`).
- **Подвал** (⏱ аптайм · ▣ версия прошивки) под списком — **без изменений**.
- Кастомный header («Настройки» + «Закрыть») и `.toolbar(.hidden, for: .navigationBar)` —
  без изменений. Поведение `WheelParamsView`/`CalibrationView` и мандатори-мастера — не
  трогаем.

## Локализация

Новые ключи (без кириллицы в Swift, всё через `L`):
```
"settings.groupSetup"   = "Настройка машины";
"settings.groupDriving" = "Движение";
"settings.groupSystem"  = "Система";
```
Аксессоры: `L.settingsGroupSetup` / `L.settingsGroupDriving` / `L.settingsGroupSystem`.

## Проверка

UI-only, юнит-тестировать нечего. Гейт — компиляция и глаз:
- `cd ios && xcodegen generate && xcodebuild build` → `BUILD SUCCEEDED`.
- Скриншот в симуляторе (мок запущен): три секции с заголовками, иконка-руль у «Колесо и
  моторы», подвал на месте.

## Вне объёма

- Любые изменения самих экранов-детей (WheelParams, Калибровка, Разгон и т.д.).
- Переупорядочивание/переименование пунктов сверх указанного.
- Сворачиваемые секции, поиск, иное — YAGNI.
