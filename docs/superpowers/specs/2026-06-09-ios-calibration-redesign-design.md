# iOS: редизайн экрана калибровки — Design

**Дата:** 2026-06-09
**Статус:** дизайн утверждён (через визуальный компаньон), готов к плану

## Цель

Сделать экран калибровки симпатичнее и понятнее: тёплая тема, нормальная диаграмма машинки,
сплит-раскладка, наглядные состояния колёс и прогресса. Логика и REST-протокол не меняются —
только вид `CalibrationView`.

## Решения (выбраны в визуальном компаньоне)

| Аспект | Решение |
|---|---|
| Раскладка | **Сплит B**: слева — машинка (крупно), справа — шаги/инструкция/действия |
| Прогресс | **4 сегмента** сверху правой панели (пройденные/текущий = зелёные, текущий со свечением) |
| «Крутится» | пульс-кольцо вокруг машинки в состоянии идентификации (после Spin, до тапа) |
| Выбор направления | **пилюли в правой панели**: `↑ вперёд` (зелёная) / `↓ назад` (янтарная) после тапа колеса |
| Колёса | серый (`idleWheel`) = ждёт · янтарь (`warn`, свечение) = выбрано/крутится · зелёный ✓ (`accent`) = размечено |
| Тема | тёплая (`Palette`), SF Symbols на кнопках |

## Раскладка (ландшафт)

`HStack`:
- **Левая половина:** диаграмма машинки сверху — кузов (вытянутый, `panel`/`line`) + 4 колеса-кнопки
  по углам (FL/FR/RL/RR), увеличенная (~×1.5). Вокруг — пульс-кольцо (`warn`, мягкая анимация),
  видимое пока `pending == nil` в активном шаге (идёт идентификация).
- **Правая половина (столбец):**
  - сегмент-прогресс (4 полоски);
  - «Шаг N из 4»;
  - **substate-блок:**
    - `pending == nil` (после Spin): инструкция «Крутится мотор N — тапни колесо, которое поехало» +
      ряд кнопок **Spin** (`play.fill`, primary) и **Save** (`checkmark`, активна при step==4);
    - `pending == c` (тапнул колесо): «Колесо C — куда крутилось?» + пилюли **↑ вперёд** (`arrow.up`,
      accent) / **↓ назад** (`arrow.down`, warn).

## Поведение (без изменений логики)

- Состояние/действия те же, что сейчас: `step`, `assign: [Corner:(pair,sign)]`, `pending`, `saving`;
  `CalibClient.spin(pair:step,dir:1)`, `save(body: ControlModel.calibSaveBody(assign))`.
- Колесо: тап только у неназначенного; назначенные — ✓ и недоступны; `pending` подсвечивается янтарём.
- Save активна при `step == 4`; успех → `dismiss()`.

## Компоненты

| Файл | Изменение |
|---|---|
| `ios/ESP32Car/CalibrationView.swift` | переписать `body` на сплит-раскладку (машинка слева, шаги/пилюли справа, сегмент-прогресс, пульс-кольцо). Логика/`CalibClient`/`calibSaveBody` — без изменений |

Прошивка/мок/протокол не трогаются. Новых файлов нет (всё в `CalibrationView.swift`; при желании
вынести машинку-диаграмму в приватный под-вью того же файла).

## Тестирование

- **Сборка:** `xcodebuild` под симулятор-SDK.
- **Симулятор (против мока, `calibrated=false`):** авто-лист → сплит-вид; сегменты заполняются по шагам;
  Spin → пульс + лог `calib/spin`; тап колеса → янтарь + пилюли направления; выбор → ✓, шаг++; после 4 →
  Save активна → `calib/save` → закрытие; обе темы.
- Чистой логики новой нет (`calibSaveBody` уже покрыт тестом).

## Вне объёма

- Изменение протокола `/calib*`, логики назначения, прошивки.
- Анимация конкретного «правильного» колеса при Spin (физически неизвестно до идентификации) — только
  общий пульс вокруг машинки.
