# Редизайн веб-пульта (`index.html`) — Design

**Дата:** 2026-06-08
**Статус:** дизайн утверждён (через визуальный компаньон), готов к плану реализации

## Цель

Переработать веб-интерфейс машинки (`main/web/index.html`, который отдаёт HTTP-сервер
и captive-портал): лаконичный, аккуратно спозиционированный, современный вид; продумать
все элементы и переходы между состояниями. **Прошивка не меняется** — те же `/ws`
(текст `t,y`, поток 10 Гц), `GET /calib`, `POST /calib/spin`, `POST /calib/save`.

## Решения (выбраны в визуальном компаньоне)

| Аспект | Решение |
|---|---|
| Раскладка | **B — ландшафт, геймпад**: элементы управления в нижних углах под большие пальцы |
| Стиль | **Stealth / минимал**: почти чёрный фон (`#0a0a0a`), тонкие серые линии (`#1f1f1f`–`#262626`), много воздуха |
| Акцент | зелёный `#4ade80` (совпадает с «брендом»; задаётся одной CSS-переменной) |
| Танк | два вертикальных джойстика в нижних углах (левый/правый борт) |
| Аркада | джойстик справа + тонкий индикатор газа слева (зеркалится для левшей) |
| Калибровка | **вид машинки сверху**: тап по колесу, что крутанулось, + направление |
| Переходы | мягкий кросс-фейд ~200 мс между экранами |
| Связь | плашка `connecting`/`connected`/`reconnecting`; при потере связи стики гаснут |
| Типографика | системный шрифт (`-apple-system, system-ui`) |

## Архитектура (один файл, без прошивки)

Переписываем `main/web/index.html`. Логика сохраняется, меняются разметка/стиль/UX.
**Сохранить как есть:**
- Математику схем: аркада `t=-stickY, y=stickX`; танк `t=(L+R)/2, y=(L-R)/2`.
- Поток `t,y` 10 Гц по WebSocket (`setInterval` 100 мс), `release → 0,0`.
- Запоминание схемы в `localStorage`.
- Гейтинг по `GET /calib`; протокол `/calib/spin "<pair>,<dir>"`, `/calib/save
  "<p>:<s>,<p>:<s>,<p>:<s>,<p>:<s>"` (порядок FL,FR,RL,RR).
- PWA-меты + иконку в `<head>` (уже есть).

**CSS:** один блок с переменными (`--bg`, `--panel`, `--line`, `--accent`, `--muted`).
Ориентация — оптимизирована под **landscape**; в portrait показываем мягкую подсказку
«поверни телефон» (страница остаётся рабочей).

## Экран «Пульт» (landscape)

- **Сверху по центру** — плашка статуса: точка + текст (`connecting…`/`connected`/
  `reconnecting…`) и компактный сегментированный переключатель **Arcade / Tank**.
- **Танк:** два круглых джойстика в нижних углах. База `#141519` + бордюр `#222`,
  кнопка-«грибок» `--accent`. Вертикальная ось → скорость борта.
- **Аркада:** один джойстик в правом нижнем углу (обе оси: вверх=газ, влево/вправо=поворот);
  слева тонкая вертикальная **шкала газа**, заполняется от центра вверх/вниз по `throttle`
  (живая обратная связь). Кнопка зеркалирования (лево/право) — опционально.
- **Маленькая «⚙»** (перекалибровать) — ненавязчиво в углу.
- **Потеря связи:** база/грибок стиков → `opacity .35`, плашка красная, поток ставится
  на паузу до `onopen`.

## Экран «Калибровка» (вид сверху)

- Заголовок: «Какое колесо крутится? Шаг N/4».
- По центру — **схема машинки сверху** (кузов + 4 колеса по углам, лёгкий намёк на лобовое
  стекло = «нос»). Уже назначенные колёса помечены галочкой и не тапаются; оставшиеся —
  активны.
- **«▶ Spin»** — `POST /calib/spin "<pairIndex>,1"` (крутит текущую пару ~0.6 с).
- Пользователь смотрит, **какое колесо крутанулось**, и **тапает его** на схеме →
  затем выбирает направление **↑ forward / ↓ back** (вперёд/назад относительно носа) →
  это задаёт `sign` (+1/−1). Переход к следующей паре.
- После 4 пар (все углы назначены) — **«✔ Save & Drive»** → `POST /calib/save` в порядке
  FL,FR,RL,RR → при успехе кросс-фейд на пульт.
- Внутреннее соответствие: шаг i гоняет `pairIndex = i`; назначение угла = `corner → {pair:
  pairIndex, sign}`. (Логика та же, что в текущем визарде, UI — пространственный.)

## Состояния и переходы

```
Загрузка (fetch /calib)
   ├─ calibrated=false → Калибровка ──(save ok)──▶ Пульт
   └─ calibrated=true  ───────────────────────────▶ Пульт
Пульт ──(⚙)──▶ Калибровка
```
- Один контейнер, секции `#calib` / `#drive` переключаются классом `.hidden` с
  CSS-`transition: opacity .2s` (кросс-фейд).
- Плашка связи реагирует на `ws.onopen/onclose`; `reconnecting` + авто-reconnect (1 с).

## Тестирование

- **Внешний вид/раскладка/переходы** — открыть `192.168.4.1` на телефоне (landscape),
  визуально сверить с макетами; проверить аркаду (стик+шкала), танк (два стика),
  переключатель, гашение стиков при обрыве.
- **Калибровка** — пройти мастер «вид сверху»: Spin → тап колеса → направление → Save →
  убедиться, что пульт рулит правильными колёсами; переживает перезагрузку.
- **Регресс** — `t,y` поток 10 Гц, гейтинг, протокол `/calib*` не сломаны.

## Открытые параметры (не блокируют)

- Зеркалирование аркадного стика (лево/право) — включать сразу или отложить.
- Подсказка «поверни телефон» в portrait — текст/иконка.
- Точные размеры стиков под разные экраны (подбор на устройстве).

## Вне объёма

- Изменения прошивки (эндпоинты те же).
- Телеметрия скорости (прошивка ничего не шлёт назад; «шкала газа» считается на клиенте
  из текущего `throttle`, не с машинки).
- Нативное iOS-приложение — отдельная спека `2026-06-08-ios-app-design.md`.
