# Симуляция траектории трюка (v1 — «Пончик»)

**Дата:** 2026-06-17
**Статус:** утверждён, готов к плану

## Идея

Зная мотор (номинальные об/мин), диаметр колеса и таймлайн трюка, прогнать кинематику
танкового (дифференциального) привода и показать **реальную траекторию** машинки сверху +
**область**, которую заметает корпус (25×15 см), + **сколько проедет** за отведённое время
(путь, обороты) — сразу видно, успеет ли «Пончик» сделать круг(и) или нет.

**v1 — только трюк «Пончик»** (`Tricks.donut`, id 4). Модель `TrickSim` пишем общую (та же
математика для любого трюка), но в UI включаем и проверяем пока только на пончике; остальные
трюки — без изменений. Чисто **iOS, без прошивки** (переиспользуем `/wheel` + `MotorPresets`).

## Модель — `ios/ESP32Car/TrickSim.swift` (чистая, host-тест)

```swift
struct TrickSim {
    struct Pose { let x: Double; let y: Double; let theta: Double }   // метры, радианы
    struct Result {
        let poses: [Pose]        // даунсэмпл для рисования/анимации (центр корпуса)
        let pathLenM: Double     // длина пути центра, м
        let turnRad: Double      // суммарный |поворот|, рад → обороты = turnRad / 2π
        let areaWM: Double       // габарит заметаемой области по X, м
        let areaHM: Double       // габарит по Y, м
    }
    // vmaxMS = π·D·(об/мин)/60 ; trackM, carLenM, carWidM — м
    static func simulate(steps: [TrickStep], vmaxMS: Double, trackM: Double,
                         carLenM: Double, carWidM: Double, dtMS: Int = 10) -> Result
}
```
Интегрирование: для каждого шага по `dtMS`: `(l,r)=ControlModel.sides(t,y)` →
`vL=l·vmax, vR=r·vmax` → `v=(vL+vR)/2`, `ω=(vR−vL)/track` → `θ+=ω·dt; x+=v·cosθ·dt;
y+=v·sinθ·dt`. Копим `pathLen += |v|·dt`, `turn += |ω|·dt`; габариты — по 4 углам корпуса
(центр + поворот `(±L/2,±W/2)` на θ) min/max за весь путь. Поза сэмплится (напр. каждые ~30 мс).

**Константы корпуса (v1, хардкод; TODO — в настройки к моторам):** `carLen=0.25 м`,
`carWid=0.15 м`, `track=0.13 м` (≈ ширина).

**Об/мин и диаметр:** диаметр — из `/wheel` (`WheelClient`), об/мин — из
`MotorPresets.match(ppr,gearX100,quad)?.rpm`. Нет совпадения («Свои параметры») → скорость
неизвестна → симуляция показывает заглушку «выберите модель мотора» (без чисел).

**Host-тесты (swiftc):** прямая (`t=1,y=0`, 1 с, vmax=1, track=0.15) → `pathLen≈1.0`,
`turn≈0`; чистый разворот на месте (`t=0,y=1`) → `pathLen≈0`, `turn>0`; пончик (`t=0.7,y=1`)
→ `pathLen>0` и `turn>0`, радиус ≈ `0.35·track` (проверить кривизну/возврат к старту).

## Левая панель — `ios/ESP32Car/TrickSimView.swift`

`Canvas` + `TimelineView(.animation)`, авто-цикл (как на других экранах):
- **Заметаемая область** — бледные «призраки» корпуса (полупрозрачные прямоугольники 25×15
  в позах вдоль пути) → читается как залитая область.
- **Траектория** — зелёный пунктир (полилиния центров).
- **Габаритная рамка** — пунктир вокруг области + подписи в **см**: ширина сверху, высота
  сбоку (как на чертеже).
- **Машинка** — корпус 25×15, едет/крутится по пути: фаза `= (t mod totalSec)/totalSec` →
  индекс в `poses` → поза + ориентация θ.
- Авто-масштаб: вписать габаритную рамку с отступом в доступный размер.

Под симуляцией — **3 чипа**: «Путь» (м) · «Оборотов» (turnRad/2π) · «Область» (Ш×В см) +
строка-вердикт: «За N с: K полных кругов + r» (`K=floor(оборотов)`, `r=дробь`).

## Локализация (без кириллицы в Swift)

Новые ключи (значения чисел подставляем форматированием в Swift):
- `sim.path` = «Путь», `sim.turns` = «Оборотов», `sim.area` = «Область» — подписи 3 чипов.
- значения чипов: путь `"%.1f м"` (`sim.pathVal`), обороты `"%.1f"` (просто число),
  область `"%d×%d"` см.
- `sim.verdict` = `"За %.1f с — %.1f оборота"` (`sim.verdict(sec, turns)`), один аргумент-секунды
  и обороты дробью; завершение читается из дробной части (напр. «За 5.0 с — 3.6 оборота»).
- `sim.pickMotor` = «Выберите модель мотора для расчёта».
- единицы `unit.m` («м»), `unit.cm` («см»).

## Допущения / вне объёма

- Модель **идеализированная, без проскальзывания** — реальный пончик из-за заноса шире; это
  прогноз-ориентир (трюки и так open-loop, без гироскопа).
- v1 — **только пончик**; остальные трюки и адаптивные подписи («Циклов» для линейных) — потом.
- Редактируемые габариты, расчёт скорости на борту (PCNT) — отдельные будущие задачи.
