# iOS: кнопка ⚙ + экран настроек (заглушка калибровки) — Design

**Дата:** 2026-06-08
**Статус:** дизайн утверждён, готов к плану

## Цель

Добавить вход в настройки/калибровку: кнопка-шестерёнка `⚙` в верхней панели пульта рядом с
переключателем режима, открывающая экран настроек со строкой **«Калибровка»** (пока заглушка —
реальный мастер калибровки будет отдельным шагом).

## Решения

- **Кнопка `⚙`** — в топ-баре `DriveView`, **справа от** переключателя Arcade/Tank. Иконка
  `gearshape` (SF Symbol), цвет `palette.muted`, фон `palette.panel`, бордер `palette.line`,
  скруглённая; обе темы. Тап → открывает настройки.
- **`SettingsView`** — презентуется как `.sheet(isPresented:)`:
  - заголовок «Настройки»;
  - строка **«Калибровка»** (иконка `gearshape`/колёса + шеврон `chevron.right`) — **заглушка**:
    тап ведёт на простой `CalibrationView` с текстом «Калибровка — в разработке» (и кнопкой назад);
  - кнопка **«Закрыть»** (dismiss).
  - Тема — из `palette` через `@Environment(\.colorScheme)`, следует системной.

## Архитектура (файлы)

| Файл | Изменение |
|---|---|
| `ios/ESP32Car/SettingsView.swift` *(new)* | экран настроек (sheet) + строка «Калибровка» → стаб `CalibrationView` |
| `ios/ESP32Car/DriveView.swift` | кнопка `⚙` в топ-баре (справа от `SchemeToggle`) + `@State showSettings` + `.sheet` |

## Тестирование

- **Сборка:** `xcodebuild` под симулятор-SDK.
- **Симулятор:** кнопка `⚙` видна справа в топ-баре; тап → лист настроек; «Калибровка» →
  «в разработке»; «Закрыть» закрывает; обе темы.

## Вне объёма

- Реальный мастер калибровки (spin → тап колеса → направление → save через `/calib*`) — отдельная
  фаза (web-аналог в `main/web/index.html`; спека iOS-калибровки — будущее).
- `/calib` в mock-сервере — не нужен для заглушки.
- Прошивка не трогается.
