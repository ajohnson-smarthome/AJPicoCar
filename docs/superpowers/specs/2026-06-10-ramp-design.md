# Ramp (плавный разгон) — Design

**Дата:** 2026-06-10
**Статус:** дизайн утверждён, готов к плану

## Цель

Гасить рывки и пусковой ток (брауны-аут 5–10× номинала, готча #6): PWM растёт плавно к цели,
падает мгновенно (безопасный стоп). Время разгона настраивается из iOS-аппа и хранится в NVS.

## Прошивка — ядро

- **`main/ramp.{c,h}`** *(new)*:
  - **Чистая** `uint16_t ramp_step(uint16_t current, uint16_t target, uint16_t max_up)` —
    рост ограничен `max_up` за вызов, падение мгновенно (`target < current` → сразу `target`).
    Zero-IDF-deps, хост-тест в `test/`.
  - Ramp-таск **50 Гц** (FreeRTOS task, период 20 мс): шагает `s_current[8] → s_target[8]`
    и пишет PCA9685. **Единственный писатель в I2C** после старта.
- **`main/car.c`**: `car_drive`/`car_spin_pair`/`car_stop` больше не пишут в PCA9685 —
  кладут целевые duty в `s_target[8]` под мьютексом (`motors_apply` заменяется на запись цели).
  Мьютекс защищает `s_target` + `g_cfg` (read и write, как после аудита).
- **Параметр**: `ramp_ms` — время разгона 0→4095 в мс; `0` = ramp выключен (мгновенно).
  `max_up = ramp_ms == 0 ? 4095 : max(1, 4095 / (ramp_ms / 20))`. Дефолт **300 мс**. Диапазон 0..2000.
- **Безопасность**: стоп/watchdog (`car_stop`) кладут 0 в цель → падение применяется в тот же тик
  (≤20 мс). Рост никогда не обгоняет цель.

## Персистентность + API

- `ramp_ms` в **NVS** (тот же неймспейс, что калибровка; ключ `ramp_ms`), грузится в `car_init`,
  применяется к ramp-таску.
- REST на общем httpd (`main/ramp_api.{c,h}` или внутри ramp.c — по месту):
  - **`GET /ramp`** → `{"ramp_ms":300}`
  - **`POST /ramp`**, body `"300"` → валидация 0..2000 → применить + сохранить в NVS → `ok`.

## iOS-апп

- ⚙ Настройки → секция **«Разгон»**: слайдер 0–1000 мс (шаг 50), подпись «Плавный старт: NN мс»
  (0 → «Выкл»). `GET /ramp` при открытии, `POST /ramp` при отпускании слайдера (`onEditingChanged`).
  Строки через `L` (`ramp.*`). Клиент — маленький `RampClient` или расширение `CalibClient`-паттерна.
- **Мок** `tools/mock_car/mock_car.py`: `GET/POST /ramp` (в памяти, дефолт 300).

## Тестирование

- **Хост-тест** `test/test_ramp.c`: рост ограничен (`current+max_up`), точное достижение цели
  (не перешагивает), падение мгновенно, `max_up=4095` (ramp off) — один шаг, граничные 0/4095.
- Сборка прошивки `idf.py build`; все host-тесты.
- Симулятор: слайдер читает/пишет мок; обе темы.
- **Бенч (пользователь)**: подбор `ramp_ms` слайдером — мягкий старт без браунаута; стоп мгновенный;
  одиночные консольные `mix` доезжают до полной скорости (ramp-таск довозит цель — фикс прошлой блокады).

## Вне объёма

- Плавное торможение (решено: мгновенно — безопасность).
- Консольная команда `ramp` (апп покрывает); экспо-кривые/профили скорости (отдельные идеи #2–3).
