# Управление диаметром пончика

**Дата:** 2026-06-17
**Статус:** утверждён, готов к плану

## Идея

Сделать диаметр круга «Пончика» настраиваемым прямо в редакторе трюка. Слайдер «Диаметр
круга» (см) обратно решает соотношение скоростей бортов → команду `(t, y)`, которую
используют **и симуляция, и реальный стрим трюка** по `/ws`. Радиус пончика зависит ТОЛЬКО
от соотношения бортов и колеи (не от скорости мотора), поэтому диаметр в см вычисляется
детерминированно. Чисто **iOS, без прошивки** (продолжение пончик-симуляции).

## Математика (чистая, host-тест)

Прямая (уже в `TrickSim`): для бортов `(l, r)` и колеи `T` радиус пути
`R = T·(l + r) / (2·(l − r))`.

Обратная (новая): быстрый борт держим на полной (`l = 1`, макс. скорость), для целевого
радиуса `R` медленный борт:
```
r = (2R − T) / (2R + T)
```
команда: `t = (1 + r) / 2`, `y = (1 − r) / 2`. Зажимаем `r ∈ [0, 0.9]` — оба борта вперёд,
трюк остаётся кругом (не переходит в разворот/почти-прямую).

Колея `T = 0.13 м` — **единая константа** с симуляцией (`TrickSim`/`TrickSimView`), чтобы
сантиметры на слайдере совпадали с нарисованным кругом.

**Пример:** диаметр 50 см → R=0.25 → r=(0.5−0.13)/(0.5+0.13)=0.587 → `t≈0.79, y≈0.21`
(≈ текущий захардкоженный пончик).

## Диапазон/дефолт

Слайдер **20–150 см**, шаг 5 см, дефолт **50 см** (≈ текущий пончик). 50 см ↔ r≈0.59;
20 см ↔ r≈0.21; 150 см ↔ r≈0.84. Все в `[0, 0.9]`.

## Поток данных

- Хранение: диаметр пончика (см) — в `UserDefaults` через `TrickSettings` (новый ключ,
  напр. `trick.donut.diaCm`), рядом с длительностями. Дефолт 50.
- Шаг пончика строится из диаметра двумя чистыми хелперами:
  - `TrickSim.donutSides(diameterCm:trackM:) -> (t: Double, y: Double)` — обратный солвер
    (формула выше, клампинг `r`), host-тестируемый;
  - `Tricks.donutTrick(diameterCm:) -> Trick` — собирает донат-`Trick` (сохраняя
    `id/nameKey/icon`) с единственным шагом `(t, y)` из `TrickSim.donutSides` и `ms` базы;
    реальную длительность накладывает `Tricks.withDurations` как сейчас.
- **И `TrickSimView`, и стрим трюка в `DriveView`** берут шаг пончика из хранимого диаметра
  (вместо константы `(0.8, 0.2)`). Константа уходит; «база» пончика = диаметр 50 см.

## UI

Редактор «Пончика» (уже `ScrollView` с анимацией + статами + слайдером длительности)
получает **второй контрол** — карточку «Диаметр круга — NN см» со слайдером (20–150, шаг 5),
над слайдером длительности. При перетаскивании симуляция перерисовывается вживую (круг
растёт/сжимается, статы пересчитываются). Сброс — как у длительности (↺ к дефолту 50).

## Локализация

Новые ключи: `sim.diameter` («Диаметр круга»), значение — `"%d %@"` см (переиспользуем
`unit.cm`). Без кириллицы в Swift.

## Host-тест

Чистое: `donutSides(diameterCm:trackM:)` — round-trip (диаметр → (t,y) → `TrickSim.simulate`
→ радиус пути ≈ диаметр/2, в пределах допуска); клампинг краёв диапазона; дефолт 50 см даёт
`t≈0.79, y≈0.21`.

## Вне объёма

- Геометрия для других трюков (только пончик).
- Редактируемая колея (берём константу 13 см; позже — в настройки к моторам вместе с
  габаритами).
- Прошивка/мок — без изменений.
