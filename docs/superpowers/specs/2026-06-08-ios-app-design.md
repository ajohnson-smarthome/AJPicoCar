# Нативное iOS-приложение для ESP32-Car — Design

**Дата:** 2026-06-08
**Статус:** дизайн утверждён, готов к плану реализации

## Цель

Нативное SwiftUI-приложение для iPhone, управляющее машинкой `ESP32-Car`: нативные
тач-джойстики + haptics + поддержка игрового контроллера, плюс калибровка — всё
поверх **существующей прошивки без изменений** (WebSocket `ws://192.168.4.1/ws`
с текстом `t,y`, REST `/calib*`).

## Ключевые решения (из брейншторма)

| Вопрос | Решение |
|---|---|
| Зачем натив | авто-подключение к WiFi*, нативное управление (джойстики/haptics/геймпад), app-feel |
| Тулчейн | **бесплатный сайдлоад** (Xcode + personal Apple ID), переподписка раз в 7 дней |
| Авто-WiFi | **НЕ делаем** — `NEHotspotConfiguration` требует платного аккаунта ($99/год). Подключение к сети — вручную, приложение детектит и подсказывает |
| Объём | езда **и** калибровка (паритет с веб-пультом) |
| Подход | полностью нативное SwiftUI (вариант A), **исполнять фазами** |
| Сборка проекта | **XcodeGen** (`project.yml` → `.xcodeproj`); Xcode только для подписи/установки на телефон |
| Прошивка | **без изменений** |

\* авто-WiFi оказался платным → выпал из объёма; остальные мотивации сохранены.

## Архитектура

Новый проект в этом же репозитории, папка `ios/`. Слои с чёткими границами:

- **`CarConnection`** (`ObservableObject`) — сетевой слой. Владеет
  `URLSessionWebSocketTask` к `ws://192.168.4.1/ws`; публикует состояние
  (`disconnected` / `connecting` / `connected`); таймер 10 Гц отправляет текущее
  `t,y` (как веб-пульт — поток кормит watchdog прошивки); проверяет доступность
  `192.168.4.1`. Также делает REST-вызовы калибровки (`URLSession`).
  - Интерфейс: `connect()`, `disconnect()`, `setCommand(throttle:yaw:)`,
    `@Published state`. REST: `checkCalibrated()`, `spin(pair:forward:)`,
    `saveCalibration(_:)`.
- **`ControlModel`** — сводит ввод (тач-джойстик ИЛИ геймпад) к `(throttle, yaw)`.
  Чистая математика схем (**юнит-тестируется**, host):
  - аркада: `t = -stick.y`, `y = stick.x`;
  - танк: `t = (L + R) / 2`, `y = (L - R) / 2`.
  Хранит активную схему (`@AppStorage`).
- **Вью (SwiftUI):**
  - `ConnectView` — статус связи + инструкция ручного подключения.
  - `DriveView` — джойстик(и) + индикатор геймпада; читает `ControlModel`,
    толкает в `CarConnection`.
  - `JoystickView` — переиспользуемый drag-стик, выдаёт `(x,y)∈[-1,1]`.
  - `CalibrateView` — мастер калибровки (Фаза 2).

## Подключение (без авто-WiFi)

Приложение не может переключать WiFi, поэтому:
1. Пытается достучаться до машинки (открыть WS / `GET /calib`).
2. Доступно → «Подключено к машинке», переход к пульту.
3. Недоступно → экран «Открой Настройки → Wi-Fi → **ESP32-Car** (пароль
   `drive1234`)» + кнопка открыть Настройки (`UIApplication.open` системного URL;
   прямой диплинк в Wi-Fi-панель на современных iOS ненадёжен → открываем Настройки
   и/или показываем инструкцию). Авто-повтор проверки доступности.

⚠️ **Обязательно в Info.plist (иначе локальная сеть не заработает):**
- `NSLocalNetworkUsageDescription` — обращение к локальному IP вызывает системный
  запрос «разрешить доступ к локальной сети»; без ключа — отказ.
- `NSAppTransportSecurity → NSAllowsLocalNetworking = true` — `http://`/`ws://` к
  локальному IP блокируются ATS по умолчанию.

## Управление (Фаза 1)

- **Тач-джойстик**, аркада по умолчанию: `JoystickView` (drag «пальца» в круге) →
  `(x,y)`. Переключатель схемы на **танк** (два вертикальных стика = левый/правый
  борт). Схема в `@AppStorage`.
- **Геймпад** (`GameController`): PS/Xbox/MFi. Левый стик → аркада, либо два стика
  → танк. При подключении контроллера — индикатор «геймпад подключён»; его ввод
  имеет приоритет над тачем.
- **Haptics** (`CoreHaptics`): лёгкая отдача на нажатие/отпускание (полировка;
  деградирует тихо, если устройство не поддерживает).
- Всё сводится к `(t,y)` → `CarConnection.setCommand` → поток 10 Гц по WS.

## Калибровка (Фаза 2)

Нативный аналог веб-визарда. REST к прошивке:
- `GET /calib` → `{"calibrated":bool}`.
- `POST /calib/spin` body `"<pair>,<dir>"` (dir 1=вперёд) — крутит пару ~0.6с.
- `POST /calib/save` body `"<p>:<s>,<p>:<s>,<p>:<s>,<p>:<s>"` (порядок FL,FR,RL,RR).
Поток: 4 шага (крутить пару → отметить угол FL/FR/RL/RR + направление) → save →
переход к пульту. Гейтинг: при старте `checkCalibrated()`; если нет — мастер.

## Фазы реализации

- **Фаза 1 — Нативный пульт:** XcodeGen-проект (Info.plist с Local Network + ATS,
  free signing) → `CarConnection` (WS + 10 Гц поток + reachability) → `ConnectView`
  → `ControlModel` + `JoystickView` (аркада/танк) → геймпад → haptics. **Едет.**
- **Фаза 2 — Нативная калибровка:** REST-методы в `CarConnection` + `CalibrateView`
  + гейтинг.

## Тулчейн и разделение труда

- **Сборка проекта:** `XcodeGen` (`brew install xcodegen`); `ios/project.yml` →
  `xcodegen generate` → `ios/ESP32Car.xcodeproj`. Проект воспроизводим из YAML.
- **Я (VSCode/CLI):** пишу весь Swift, Info.plist, `project.yml`, ассеты;
  собираю под **iOS Simulator** (`xcodebuild -scheme ... -destination 'platform=iOS
  Simulator,...'`) — проверяю компиляцию и работу UI/джойстиков (без машинки).
- **Пользователь (Xcode + iPhone):** открывает `.xcodeproj`, ставит свой Apple ID
  в Signing & Capabilities (personal team), втыкает iPhone → Run (установка на 7
  дней), тестирует **реальную езду** на сети `ESP32-Car`. Переустановка раз в неделю.
- ⚠️ Симулятор **не достучится** до реальной машинки (она — WiFi-точка). Поэтому:
  компиляция + UI проверяются в симуляторе (мной), реальное вождение — на телефоне
  (пользователем).

## Тестирование

- **Юнит-тесты (host/Simulator):** чистая математика `ControlModel` — аркада/танк
  маппинг `(stick) → (t,y)`, клэмп, нормализация. Зеркалит инверсию прошивочного
  миксера.
- **Сборка:** `xcodebuild` под симулятор — компиляция без ошибок.
- **Ручное на устройстве:** подключение к `ESP32-Car`, езда джойстиком/геймпадом,
  калибровка; проверка watchdog (увести телефон/закрыть приложение → стоп).

## Прошивка

Не меняется. Приложение использует существующие эндпоинты: `ws://192.168.4.1/ws`
(текст `t,y`, поток ~10 Гц), `GET /calib`, `POST /calib/spin`, `POST /calib/save`.

## Открытые параметры (не блокируют старт)

- Bundle ID (например, `com.adamjohnson.esp32car`).
- Deployment target (iOS 16+).
- Чувствительность/мёртвая зона джойстика (подбор на устройстве).
- Раскладка геймпада под танк (два стика vs стик+триггеры).

## Вне объёма

- **Авто-подключение к WiFi** (`NEHotspotConfiguration`) — требует платного аккаунта.
- TestFlight / App Store — требует платного аккаунта.
- Телеметрия с машинки (прошивка сейчас ничего не шлёт назад по WS).
